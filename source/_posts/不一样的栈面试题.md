群里有人发过一页面试题，其中第二道面试题比较有意思。

```text
给定一个递增的整数数组，请编写一个函数，查找出该数组中不重复的元素，并给出复杂度。
例如：[1,1,2,2,3,4,5,5,5]，输出为[3,4]，不限编程语言。
```
一般以为用栈只能消除偶数的重复元素，但真的是这样吗？   
这道题是有序数组，还是可以用栈解决了这个问题。
```js
const arr = [1,1,2,2,3,4,5,5,5];

function fun(arr,i=0){

  if( i < (arr.length - 1) ){
    //没到末尾
    const next = i + 1
    const before = i - 1
    const stak = fun(arr,next);

    if(arr[i] === stak[0]){
      //本元素和返回栈顶相同

      /***
       * 开始消除逻辑
       * 我是递归到最末尾才开始往前判断的。
       * 入栈 还是普通的判断，原数组当前元素和栈顶不同就入栈。
       * 
       * 出栈 是原数组当前元素和栈顶元素相同时，
       * 还要判断原数组前一位元素是否和当前元素相同：（这是精华）
       *    相同就直接返回栈，当做已经消除本位的重复元素。
       *    不同就当普通处理，消除栈顶元素。
       */
      if(i === 0){
        stak.shift()
        return stak;
      }
      if( arr[i] === arr[before] ){
        return stak;
      }else{
        stak.shift()
        return stak;
      }
      /**
       * 结束消除逻辑
       */
      
    }else{
      //本元素和返回栈顶不同
      stak.unshift(arr[i])
      return stak
    }

  }else{
    //到了末尾
    return [ arr[i] ]
  }
}
console.log(fun(arr))
```
